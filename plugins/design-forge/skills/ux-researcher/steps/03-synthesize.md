# Step 3: Synthesize Findings

Consolidate all research from Step 2 into structured findings before writing the report.

## Synthesis Process

```yaml
synthesis_process:
  - Group patterns by type
  - Compare technology options
  - Identify best practices
  - Extract implementation examples
  - Create comparison matrices
  - Develop recommendations
```

## Checklist

1. **Group patterns** -- Organize discovered patterns by category (navigation, data display, animation, etc.)
2. **Build comparison matrices** -- For any technology evaluation, create a table with columns: Technology, Best For, Complexity, Performance, Bundle Size, When to Choose
3. **Identify best practices** -- Extract the recurring themes across sources
4. **Collect evidence** -- Gather URLs to real-world implementations that demonstrate each recommendation
5. **Form primary recommendation** -- Based on the project's constraints (from Step 1), select the best-fit approach
6. **Prepare alternatives** -- Document 1-2 alternative approaches with trade-offs

## Quality Gate

Before proceeding to the report, verify you have:
- **Actionable recommendations** (not just interesting findings)
- **Technology specifics** (library names, versions, bundle sizes)
- **Real-world examples** (URLs to live implementations)
- **Performance data** (bundle sizes, load times, benchmarks)
- **Complexity estimates** (implementation time, learning curve)
- **Clear decision matrices** (when to use what)
- **Ready for design-forge** (can immediately start implementing)

If any of these are missing, go back to Step 2 and fill the gaps.
