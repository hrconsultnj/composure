## Workflow

1. Read `/composure:app-architecture` to load Next.js-specific patterns (fullstack/nextjs/).
2. Design the route structure: route groups, layouts, loading/error boundaries.
3. Implement server components by default; use `'use client'` only when interactivity is needed.
4. Use server actions for mutations. Use TanStack Query for client-side data fetching.
5. Optimize: static generation where possible, dynamic rendering only when needed.
6. Verify Core Web Vitals: LCP, FID, CLS within acceptable thresholds.
