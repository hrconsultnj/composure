# Use Cases

Real-world scenarios where Composure's features solve problems that vanilla Claude Code can't.

Each case study documents what happened, what the plugin did differently, and the measurable impact.

| Use Case | Plugin Feature | Key Metric |
|----------|---------------|------------|
| [Code Graph vs Explore Agents](graph-vs-explore/) | Code review graph, session hooks | 30x fewer tokens, 9x faster, zero missed code |
| [Multi-Project Graph at Scale](multi-project-graph/) | 8-language parsers, cross-project queries | 15 projects, 100K nodes, 90x faster discovery |
| [12-Minute Vulnerability Resolution](vulnerability-resolution/) | Sentinel + Shipyard parallel audit | 4 CVEs patched in one session, no context switch |
| [Blueprint vs Plan Mode](blueprint-vs-plan-mode/) | Graph-powered planning vs Explore agents | 2.75x fewer tokens, 2.6x faster, higher quality plans |
| [Progressive Refinement](progressive-refinement/) | Template-driven quality, native tool learning | 28/40 → 39/40 quality, one template change, same token cost |
| [Session Context Optimization](session-context-optimization/) | Lightweight hooks, on-demand loading | ~200 lines → ~8 lines at session start |
| [Graph Narrows, Agent Deepens](graph-narrows-agent-deepens/) | search_references, get_dependency_chain, JSDoc summaries | 2.85x faster, 37% better quality, neither alone is optimal |
